4 Standing Rules — Non-Negotiable
⛔ Never Do
  • ✗ SDK for data — REST only
  • ✗ card_id in vault insert (FK error)
  • ✗ JSON.stringify in onclick without .replace(/"/g,'&quot;')
  • ✗ PowerShell > to write files (UTF-16 LE)
  • ✗ Touch card-viewer.html — ever
  • ✗ Cloudflare dashboard instructions (registrar = Namecheap)
  • ✗ Remove email-decode line (~line 2192)
  • ✗ Build /admin/fomo yet
  • ✗ Fake timers or fabricated FOMO signals
✅ Always Do
  • ✓ data-cfasync="false" on all script tags
  • ✓ Node.js syntax check before every delivery
  • ✓ Read full function before touching it
  • ✓ RLS + public read on every new table
  • ✓ Double-quote SQL table names with special chars
  • ✓ _ivRealTable() for all Pokémon table fetches
  • ✓ "What time is it?" before file delivery
  • ✓ Upload PROJECT.md at session start
  • ✓ Fix ALL related issues in a single pass
⚡ Error Patterns
  • HTTP 404 → table name in IV_SETS doesn't match Supabase exactly
  • HTTP 400 → column in SELECT doesn't exist. Fix: add noVarImages:true
  • Cloudflare SyntaxError → script tag missing data-cfasync="false"
  • Vault FK error → card_id included in insert. Remove it.
  • JSON.stringify onclick → missing .replace(/"/g,'&quot;') chain
  • cd-info collapsed → cd-viewer-wrap div not closed before cd-info
  • File truncation → used PowerShell > redirect (UTF-16)
